# Integration catalog — frontend → backend touchpoints

> Generated by Task B1. Captures every place the frozen frontend talks to
> the backend, what's wired, what's stubbed, and where the shapes diverge.
> Use this as the input to subsequent polish tasks.

## Status legend
- ✅ **wired** — modal/action persists end-to-end
- 🔌 **stub** — frontend UI exists but the callback is empty (or local-only)
- 🐛 **bug** — frontend reads a field name the backend doesn't return
- ➕ **missing** — backend endpoint is needed for a frontend action
